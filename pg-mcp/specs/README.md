## 文档阅读顺序

### 新成员入门

如果你是新加入项目的成员，建议按以下顺序阅读文档：

1. **[产品需求文档 (0001)](./0001-pg-mcp-prd.md)** - 了解项目背景和目标
2. **[系统设计文档 (0002)](./0002-pg-mcp-design.md)** - 理解系统架构和设计决策
3. **[实现计划 (0004)](./0004-pg-mcp-impl-plan.md)** - 掌握开发路线图
4. **[测试计划 (0007)](./0007-pg-mcp-test-plan.md)** - 了解测试策略和质量要求

### 开发人员

开发过程中需要参考的文档：

1. **[实现计划 (0004)](./0004-pg-mcp-impl-plan.md)** - 查看当前阶段的任务和接口定义
2. **[系统设计文档 (0002)](./0002-pg-mcp-design.md)** - 查阅组件设计和接口规范
3. **[代码评审改进设计 (0009)](./0009-pg-mcp-codex-review-design.md)** - 查看当前的改进方案
4. **[测试计划 (0007)](./0007-pg-mcp-test-plan.md)** - 编写测试时参考

### 评审人员

进行代码评审或设计评审时：

1. 先阅读对应的规范文档（0001/0002/0004/0007）
2. 参考相关的评审报告（0003/0005/0006/0008）
3. 检查是否符合设计要求和最佳实践

## 文档编写规范

### 文档命名规范

- 核心文档使用 4 位编号：`0001-pg-mcp-{type}.md`
- 编号规则：
  - 0001-0099: 需求和产品文档
  - 0002-0099: 设计文档
  - 0003-0099: 评审文档
  - 0004-0099: 实现文档
  - 0007-0099: 测试文档

### 文档状态标识

- ✅ 已完成/已审查 - 文档已完成并通过评审
- 🚧 进行中 - 文档正在编写或修订中
- 📋 计划中 - 文档已规划但未开始
- 🔒 已锁定 - 文档已归档，不再修改

### 文档版本管理

每个文档应在开头包含文档信息表：

```markdown
## 文档信息

| 项目 | 内容 |
|------|------|
| 文档版本 | v1.0 |
| 创建日期 | YYYY-MM-DD |
| 最后更新 | YYYY-MM-DD |
| 状态 | 已审查/进行中/等待评审 |
| 作者 | 作者名称 |
```

## 文档维护

### 更新流程

1. 创建或修改文档时，更新文档信息表中的版本号和日期
2. 如果是重大变更，在本 README 中更新文档状态
3. 相关文档变更时，同步更新引用关系

### 归档规则

- 被新版本替代的文档移至 `archive/` 子目录
- 保留至少一个历史版本作为参考
- 重复或过时的文档在归档前需经过团队确认
